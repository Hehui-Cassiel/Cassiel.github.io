hexo的搭建

作者 He Hui 日期 2019-11-16
hexo的搭建

搭建hexo博客

申请github账号

Github注册页面

输入用户名，Email，密码，注册账号

创建博客仓库

注意，仓库名应该为：用户名.github.io。比如，我的仓库名是：hehui-cassiel.github.io。

搭建环境

1.安装下载node.js

2.安装npm

3.检验node和npm是否安装成功，在cmd中输指令：node-v 和 npm -v

搭建hexo和git

安装HEXO

使用安装语句，并检验是否成功

1.npm install -g hexo-cli
2.hexo init blog
3.hexo -v

建站

1.在终端切换（cd）一个你想指定的目录(blog)，执行下面这个命令

    hexo init blog

2.在这个目录下会出现你刚刚创建的blog文件夹，切换到（cd）blog文件夹执行以下命令来安装npm

    npm install

③执行以下命令来开启服务器

   hexo server

④开启后访问[http://localhost:4000/

(http://localhost:4000/)

与github联系起来

1.首先创造一个新的仓库，给仓库命名（示例：everything.github,io，其中everything可改变，加粗地方不可变）

2.将blog名下的_config.yml内容那个最后更改为：

deploy:
    type:git
    repository:https://github.com/username/username.github.io.git

3.在blog目录下执行命令

hexo generate//生成静文件
hexo deploy//部署网站

此时可以访问属于自己的网站了https://username.github.io/

建立博客后怎样发布

在本地文件中，用Git Bash执行

hexo n “my article”     #完整命令为hexo new,用于新建一篇名为“filename”的文章
hexo g    #完整命令为hexo generate,用于生成静态文件
hexo s    #完整命令为hexo server,用于启动服务器，主要用来本地预览
hexo d    #完整命令为hexo deploy,用于将本地文件发布到github等git仓库上

主题更换

参考链接：https://hexo.io/themes/

其实就是拿到你喜欢主题的链接，然后到blog目录下执行命令

git clone 链接地址

下载成功的主题拖到themes文件夹里




改完后输入命令:

hexo clean

hexo g

hexo d

易出现的两个错误:

第一个错误

1.检查下载的主题是不是放在了themes文件夹下面，比如next主题的放置，就是点开themes就看到next的文件夹，next下面就是主题的文件，不能打开next文件夹还有themes

2.检查站点配置_config.yml是不是下面的形式 theme: 你的主题名字
注意： （1）这里是theme，不是themes.
（2）theme后面与主题名直接有一个空格。
（3）主题名字是否与themes文件夹下 主题的名字一致。
比如next：theme: next,冒号后面有空格。 假如全部一致还运行有错，建议每次修改后 你先输入命令hexo clean清理一下原来的配置，然后hexo s --debug运行，浏览器页面打开隐身模式重新打开一个页面。

第二个错误： 本地主题样式修改完之后，本地运行访问都对，然后hexo d部署到github上面后访问没有样式。 打开站点配置_config.yml 更改其中的url和root配置 url: 这里是GitHub给你分配的地址 root: /项目名 然后重新 hexo g 接着hexo d 解决问题。
